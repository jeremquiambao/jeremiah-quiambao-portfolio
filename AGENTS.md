# AGENTS.md

## Project Overview

This repository contains Jeremiah Quiambao's portfolio website. It is a living portfolio that should improve over time as Jeremiah grows in web development and AI automation.

The site is not just a personal profile. It is intended to function as a business asset that:

- attracts clients and collaborators
- demonstrates real skill in web development, design, and automation
- builds trust quickly through clear presentation and polished execution
- converts visitors into inbound leads through the contact form

The deployment target is Vercel.

## About The Owner

Jeremiah is building this portfolio alongside a long-term journey in:

- web development
- AI automation
- client-focused digital solutions

The site should reflect someone who is actively improving, shipping work, and becoming more capable over time. Updates should strengthen credibility, clarity, and conversion.

## Long-Term Goal

The long-term goal of this project is to turn the portfolio into a strong lead-generation website that helps Jeremiah get hired, win freelance work, and attract collaboration opportunities.

Success looks like:

- the website feels attractive, modern, and intentional
- visitors quickly understand Jeremiah's skills and value
- project work is presented clearly and professionally
- the contact section works reliably and encourages outreach
- the site becomes stronger over time as new work and capabilities are added

## Current Product Direction

When making changes, prioritize work that improves one or more of these areas:

1. Visual quality
2. Clarity of messaging
3. Trust and credibility
4. Contact conversion
5. Quality of showcased work
6. Performance and deploy readiness on Vercel

## Practical Guidance For Future Agents

- Keep the site aligned with Jeremiah's positioning as a web developer and AI automation builder.
- Prefer improvements that help real visitors understand services, proof of work, and how to get in touch.
- Treat the contact form as a critical business feature, not a secondary detail.
- Preserve or improve Vercel compatibility when touching frontend or serverless code.
- Favor polished, intentional presentation over generic portfolio filler.
- Make incremental improvements that compound over time instead of random redesigns.
- When adding new sections or projects, keep the content believable, specific, and outcome-oriented.

## Near-Term Priorities

- improve the portfolio's visual appeal
- strengthen copy so it speaks to clients and collaborators
- expand showcased projects as Jeremiah's work grows
- refine the contact flow so inquiries are easy to submit
- prepare the repo for future skills, automations, and repeatable workflows

## Working Principle

Every meaningful change should answer at least one of these questions:

- Does this make the site more attractive?
- Does this make Jeremiah more credible?
- Does this make it easier for a visitor to contact him?
- Does this improve the chances of getting hired or invited to collaborate?
